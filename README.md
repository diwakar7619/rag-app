# 📄 PDF RAG Assistant

A **Retrieval-Augmented Generation (RAG)** application built with **Python**, **LangChain**, **ChromaDB**, **Google Gemini**, and **Streamlit**.

This application allows users to upload any PDF document and ask natural language questions about its contents. Instead of relying solely on the LLM's general knowledge, the application retrieves the most relevant sections of the document using vector search before generating a grounded response with Gemini 2.5 Flash.

## 🚀 Features

* 📄 Upload any PDF document
* 🔍 Semantic search using vector embeddings
* 🧩 Automatic document chunking with overlap
* 🗂️ ChromaDB vector database for fast retrieval
* 🤖 Grounded answers generated by Gemini 2.5 Flash
* ⚡ Cached vector store for improved performance
* 🌐 Interactive Streamlit web interface

## ⚙️ Installation

Clone the repository:

```bash
git clone https://github.com/diwakar7619/pdf-rag-assistant.git
```

Navigate to the project:

```bash
cd pdf-rag-assistant
```

Install dependencies:

```bash
uv sync
```

Create a `.env` file:

```env
GOOGLE_API_KEY=your_api_key_here
```

Run the application:

```bash
streamlit run app.py
```

## 🧠 How It Works

1. Upload a PDF document.
2. Extract text using **PyPDFLoader**.
3. Split the document into overlapping chunks.
4. Generate embeddings using **Gemini Embeddings**.
5. Store embeddings in **ChromaDB**.
6. Convert the user's question into an embedding.
7. Retrieve the most relevant chunks using semantic similarity search.
8. Build a context from the retrieved chunks.
9. Generate a grounded answer using **Gemini 2.5 Flash**.
